# 一、项目背景

## 1.1 现状：AI工具的"三道门槛"

你是否遇到过这些场景？

**场景1：钉钉上有人问你问题**
- 你需要：截图 → 打开AI → 上传图片 → 输入问题 → 等待回答
- 耗时：2-3分钟
- 你想要：直接问AI"他在说啥"，3秒得到答案

**场景2：Aone需求要分析**
- 你需要：看需求 → 打开Aone → 手动查关联任务 → 查负责人 → 查进度 → 自己整理
- 耗时：5-10分钟
- 你想要：AI自动查询Aone，直接给你分析报告

**场景3：监控数据异常**
- 你需要：看监控 → 截图 → 打开AI → 上传 → 再手动查历史数据 → 对比分析
- 耗时：3-5分钟
- 你想要：AI看屏幕，自动查历史，直接告诉你原因

**这就是当前AI工具的三道门槛：**

### 门槛1：看不到屏幕 🙈
- ChatGPT/Claude：需要手动截图上传
- Claude Code：不支持图片理解
- GitHub Copilot：只能看代码编辑器

**结果**：每次使用AI都要先"搬运"信息，打断工作流。

### 门槛2：无法操作企业系统 🚫
- 即使AI看到了屏幕内容
- 还是要手动打开Aone查需求
- 还是要手动打开钉钉查文档
- 还是要手动查数据库查数据

**结果**：AI只能给建议，具体操作还得你来。

### 门槛3：信息孤岛 🏝️
- 屏幕内容是一个孤岛
- 企业系统是另一个孤岛
- AI无法打通两者

**结果**：AI无法基于完整上下文做决策。

## 1.2 问题本质：AI缺少"眼睛"和"手脚"

想象一下，如果你的同事：
- 👁️ **看不到**你的屏幕（没有眼睛）
- 🤚 **操作不了**公司系统（没有手脚）
- 🧠 **不知道**上下文（没有记忆）

你会让他帮你工作吗？**不会。**

但这就是当前大部分AI工具的现状：
- ❌ 看不到你正在做什么
- ❌ 操作不了企业内部系统
- ❌ 无法基于完整上下文决策

**AI想帮你，但力不从心。**

# 二、产品介绍

## 2.1 产品理念

**"灵析"不是一个普通的AI聊天工具，而是一个能"看到"你屏幕、"操作"企业系统的智能伙伴。**

如果说AI是盲人的导盲犬，那么灵析就是AI的导盲犬。

就像真正的导盲犬能看到主人看不到的东西一样，我们的AI导盲犬能让AI理解屏幕上通过MCP获取不到的内容，并且通过MCP协议调用企业系统能力，给出分析和建议。

并且在接入了MCP以后，它可以在看到屏幕上别人提给你的问题之后，自己去数据库中查询数据并进行分析，最后把结论放在你的粘贴板中。这样可以省去你使用AI时，转述问题和复制答案的步骤和时间。

## 2.2 核心功能

**它能做什么？**

- 理解你当前的工作场景
- 基于屏幕内容回答你的问题
- 支持95%图片压缩（6MB → 300KB）

**使用场景：**

1. **看钉钉消息时**
   - 你：他在说啥
   - AI：（看下你的屏幕）他觉得今天的RAM切流结果不符合预期……
   - 你：看下合理性
   - AI：（看下你的屏幕）我们来逐层分析其合理性与潜在问题……

2. **看业务/监控数据时**
   - 你：分析下广告数据
   - AI：（看到屏幕上的业务数据）核心指标对比：曝光：xx，点击：xx，成交：xx

   - AI：（看到屏幕上的监控数据）机房na61:RT(P99/999):xx...

3. **阅读文档时**
   - 你：这啥

4. **学习新知识时**
   - 你：看下
   - AI：（看到你的教材）这个概念是指...举个例子...

**价值：**
